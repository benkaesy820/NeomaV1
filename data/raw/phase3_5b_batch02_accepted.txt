<instruction>
Implement require_json_value(value: object) -> object.
</instruction>
<constraints>
- Accept only None, bool, str, int, finite float, list, and dict with string keys.
- Validate recursively.
- Reject tuples, non-finite floats, non-string keys, and reference cycles.
- Return the original value after validation.
</constraints>
<answer>
import math


def require_json_value(value: object) -> object:
    active: set[int] = set()

    def visit(item: object, path: str) -> None:
        if item is None or isinstance(item, (bool, str, int)):
            return
        if isinstance(item, float):
            if not math.isfinite(item):
                raise ValueError(f"{path} contains a non-finite number")
            return
        if isinstance(item, (list, dict)):
            marker = id(item)
            if marker in active:
                raise ValueError(f"{path} contains a reference cycle")
            active.add(marker)
            try:
                if isinstance(item, list):
                    for index, child in enumerate(item):
                        visit(child, f"{path}[{index}]")
                else:
                    for key, child in item.items():
                        if not isinstance(key, str):
                            raise TypeError(f"{path} contains a non-string key")
                        visit(child, f"{path}.{key}")
            finally:
                active.remove(marker)
            return
        raise TypeError(f"{path} contains unsupported type {type(item).__name__}")

    visit(value, "$" )
    return value
</answer>
<reasoning>
Edge cases:
- Repeated references without a cycle are valid.
- bool is valid even though it subclasses int.
- NaN and Infinity are rejected.
Quality notes:
- Distinguishes active recursion from harmless shared references.
</reasoning>

<instruction>
Implement parseCsvRow(line) for one comma-separated record.
</instruction>
<constraints>
- Support quoted fields, commas inside quotes, and doubled quotes.
- Reject a quote that begins after unquoted text, characters after a closing quote before a comma, and an unclosed quoted field.
- Do not support embedded physical line breaks; reject CR or LF in line.

<answer>
function parseCsvRow(line) {
  if (typeof line !== "string") throw new TypeError("line must be a string");
  if (/[\r\n]/.test(line)) throw new Error("line must contain one physical record");

  const fields = [];
  let field = "";
  let quoted = false;
  let closedQuote = false;

  for (let index = 0; index < line.length; index += 1) {
    const char = line[index];
    if (quoted) {
      if (char === '"') {
        if (line[index + 1] === '"') {
          field += '"';
          index += 1;
        } else {
          quoted = false;
          closedQuote = true;
        }
      } else {
        field += char;
      }
    } else if (closedQuote) {
      if (char !== ",") throw new Error("unexpected text after closing quote");
      fields.push(field);
      field = "";
      closedQuote = false;
    } else if (char === ",") {
      fields.push(field);
      field = "";
    } else if (char === '"') {
      if (field !== "") throw new Error("quote must start at the beginning of a field");
      quoted = true;
    } else {
      field += char;
    }
  }

  if (quoted) throw new Error("unclosed quoted field");
  fields.push(field);
  return fields;

function indexRowsByField(rows, field) {

  if (typeof field !== "string" || field.trim() === "") {
